## KEY TECHNICAL DECISIONS & GOTCHAS
================================================================================

### LINE SDK imports (CRITICAL — caused multiple bugs)
  - WebhookParser  → from linebot.v3.webhook   (singular)
  - ImageMessageContent, FileMessageContent, TextMessageContent,
    MessageEvent, PostbackEvent, PostbackContent
                   → from linebot.v3.webhooks   (plural)
  - MessageEvent is shared between both modules (same class)

### Supabase async (supabase 2.31.0)
  - acreate_client / AsyncClient available
  - Async query: await client.table("receipts").select("*").execute()
  - maybe_single() returns data=None (not []) when no row found
  - Singleton pattern: module-level _client in repository.py

### Google Drive / Sheets API
  - google-api-python-client is synchronous → wrap in asyncio.to_thread()
  - Both drive.py and sheets.py handle GOOGLE_SA_JSON_B64 inline (already done)
  - Drive folder cache: dict[str, str] in drive.py, reset on process restart
  - cache_discovery=False on build() to avoid disk writes on Render

### MagicMock in tests
  - MUST use MagicMock(spec=SomeClass) for isinstance() checks
  - __class__= trick does NOT work with MagicMock

### Reply ordering (30-second rule)
  - ALWAYS: download → OCR → insert_pending → REPLY → Drive upload → Sheets sync
  - Sheets sync: happens AFTER user confirms AND after reply to confirm postback
  - Drive upload + Sheets sync are non-fatal (logged, receipt still in DB)

### Gemini SDK (google-genai 2.8.0)
  - Async path: client.aio.models.generate_content(...)
  - Vision: contents=[types.Part.from_bytes(data=..., mime_type=...)]
  - Text-only: contents=["the text string"]
  - response_mime_type="application/json" set in GenerateContentConfig

### Postback data format
  - "action=confirm&id={receipt_uuid}"
  - "action=cancel&id={receipt_uuid}"
  - "action=edit&id={receipt_uuid}"
  - Parsed with urllib.parse.parse_qsl()
  - PostbackContent.__fields__ = ['data', 'params'] (pydantic v1 style)

### Text entry synthetic message_id
  - Format: "text:{uuid4()}" — avoids collision with real LINE messageIds
  - line_message_id column has UNIQUE constraint in DB

### Edit command keywords (parse_edit_command in main.py)
  - ยอด/ยอดรวม → total_amount
  - ก่อนภาษี/ยอดก่อน → subtotal
  - ภาษี/vat/VAT → vat_amount
  - wht/WHT → wht_amount
  - หมวด/หมวดหมู่ → category
  - ร้าน/ชื่อร้าน → vendor_name
  - วันที่ → issue_date
  - ประเภท → document_type
  - เลขที่ → document_number
  - เลขผู้เสียภาษี → vendor_tax_id
  - Case-insensitive match, strips leading/trailing spaces

## DATABASE SCHEMA (create in Supabase SQL editor)
================================================================================

create table receipts (
  id uuid primary key default gen_random_uuid(),
  company_id uuid,
  line_user_id text not null,
  line_message_id text unique not null,
  status text not null default 'pending',
  vendor_name text, vendor_tax_id text, document_type text,
  document_number text, issue_date date, category text,
  subtotal numeric(14,2), vat_amount numeric(14,2),
  wht_amount numeric(14,2), total_amount numeric(14,2),
  currency text default 'THB', raw_text text, confidence real,
  drive_file_id text, drive_file_url text,
  sheet_row_synced boolean default false,
  created_at timestamptz default now(), confirmed_at timestamptz
);
create index idx_receipts_user on receipts(line_user_id);
create index idx_receipts_status on receipts(status);
create index idx_receipts_date on receipts(issue_date);
