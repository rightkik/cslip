     - GOOGLE_CLIENT_SECRET: "The provided client secret is invalid"
     - SUPABASE_SERVICE_ROLE_KEY: silent empty query results

     Preview deployments used for testing couldn't see env vars. Fixed by deploying
     directly to production with `vercel --prod` once all vars were corrected.

     Existing GOOGLE_CLIENT_ID in .env is "Desktop app" type (for Drive/Sheets refresh
     token flow). NextAuth requires "Web application" type. Created new OAuth client
     in GCP console (same project gen-lang-client-0620550147), added redirect URI:
     https://cslip-dashboard.vercel.app/api/auth/callback/google

WHY:  Each issue was a one-time setup problem, not a code bug. BOM issue is specific to
